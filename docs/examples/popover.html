Popover

A binding to display a bootstrap popover

Basic Example

This shows the default behavior for the binding and viewModel.

Demo

Binding Code

<button class="btn" title="Success" data-content="You have clicked a button.  Click it again to dismiss." data-bind="popover: simple">Click Me!</button>

ViewModel

this.simple = new Popover();

Code Configuration

This shows how the popover may be configured solely from the viewModel

Demo

Binding Code

<button class="btn" data-bind="popover: codeConfigured">Hover over Me!</button>

ViewModel

this.codeConfigured = new Popover({
    title: 'Code-Configured',
    content: 'This message was configured within the viewModel.',
    options: {
        trigger: 'hover'
    }
});

Inline Configuration

This shows how the popover may be configured from the binding

Demo

Binding Code

<button class="btn"
    title="Inline-Configuration"
    data-content="This popover was configured from a mixture of data-attributes and bindings"
    data-bind="popover: inlineConfigured, popoverOptions: { trigger: 'hover' }">Hover over Me!</button>

ViewModel

this.inlineConfigured = new Popover();

Observability

You can change the state of the observable properties of the popover to have it reflect in your view

Demo

To show popover you can hover over me or press the 'toggle' button.

Binding Code

<div class="panel panel-default" data-bind="popover: observabilityDemoPopover">
    <h4 class="panel-body">To show popover you can hover over me or press the 'toggle' button.</h4>
</div>

ViewModel

this.observabilityDemoPopover = new Popover({
    title: 'I am a title',
    content: 'I am a content.',
    options: {
        placement: 'bottom',
        trigger: 'hover'
    }
});

NOTICE

The text for title and content displayed in the popover will not change until the popover has been hidden and shown again. This is in line with the behavior that is presented in Bootstrap

Observable Template Example

This shows how you can supply a different template after the popover has been created.

NOTICE

Just like how you cannot change title and content while the popover is being displayed, you also cannot change template and have the change immediately show. Additionally, if you do change template while the popover is being displayed, the popover will be immediately closed. If you need to display the popover afterwards, it is your responsibility to do so.

Demo

Binding Code

<div class="col-xs-6">
    <button class="btn" data-bind="popover: templatePopover">Click Me!</button>
</div>
<div class="col-xs-6 text-right">
    <button class="btn" data-bind="click: setTemplate('regular'), css: { 'active': isSelected('regular') }">Regular</button>
    <button class="btn" data-bind="click: setTemplate('fancy'), css: { 'active': isSelected('fancy') }">Fancy</button>
</div>

ViewModel

this.templates = {
    'regular': '<div class="popover" role="tooltip"><div class="arrow"></div>' +
               '<h3 class="popover-title" style="font-style: italic;"></h3>' +
               '<div class="popover-content"></div></div>',
    'fancy': '<div class="popover" role="tooltip"><div class="arrow"></div>' +
             '<h2 class="popover-title" style="font-weight: bold; background-color: red; color: white;"></h3>' +
             '<div class="popover-content"></div></div>'
};

this.templatePopover = new Popover({
    title: 'Template',
    content: 'Click regular or fancy to change template styles.',
    options: {
        trigger: 'click'
    },
    template: this.templates['regular']
});

this.setTemplate = function(templateName) {
    return function() {
        self.templatePopover.template(self.templates[templateName]);
    };
};

this.isSelected = function(templateName) {
    return self.templatePopover.template() == self.templates[templateName];
}

FAQs

Q: Why is the observable property for template so bad?
A: Bootstrap doesn't really support popover templates being changed, that is a hack that this library is accomplishing by changing the bs.popover.$tip element.
